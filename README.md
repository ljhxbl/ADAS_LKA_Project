## 1. Project Overview
本项目实现了一个面向高级辅助驾驶系统（ADAS）的视觉车道保持辅助（LKA）管道。基于经典计算机视觉算法（无深度学习依赖），系统能够从前向行车记录仪视频流中实时检测左右车道边界，输出帧级置信度与横向偏移指标，并生成带有 HUD 可视化反馈的标注视频。

在 CPU 环境（Python 3.9）下，算法可达到约 **25-35 ms/帧** 的准实时处理速度。

## 2. Algorithm Pipeline
系统采用高度模块化的工程设计，单帧处理链路如下：

1. **预处理 (Preprocessing):** - 提取图像下部 55% 作为感光兴趣区 (ROI)。
   - 结合 HLS 颜色空间的 S 通道（分离车道线反光）与 L 通道的 Sobel X 梯度算子（强化垂直边缘）进行融合阈值化。
2. **逆透视变换 (IPM):** - 通过 4 点单应性矩阵（Homography）将前向视角转换为鸟瞰图 (Bird's-eye view)，使车道线近似平行，从而稳定后续的几何搜索。
3. **特征提取与拟合 (Lane Search & Fitting):** - 使用直方图峰值检测确定车道线基线点。
   - 应用滑动窗口算法（n=9, margin≈80, minpix≈50）进行局部曲线搜索。
   - 在变形空间中基于二次多项式 $x = f(y)$ 分别拟合左右车道轨迹。
4. **时序滤波与状态机 (Temporal Smoothing):** - **置信度评估:** 基于有效像素数量与拟合残差混合计算 0-1 的置信度得分。
   - **迟滞阈值:** 设定 `YES ≥ 0.65` 与 `NO ≤ 0.55` 的双重阈值状态机，有效消除视觉闪烁 (Flicker)。
   - **EMA 滤波:** 对多项式系数应用指数移动平均（EMA, $\alpha \approx 0.6$），在置信度下降时触发 `keep-last-good` 机制保持稳定。
5. **物理映射 (Lateral Offset):** - 根据标准的 3.7m 车道宽度启发式规则，将像素偏移转化为物理横向偏移量（米）。

## 3. Quantitative Performance
在不同的测试集下，系统的定量表现如下：

- **Daytime Highway (日间高速/良好光照):** - 左右车道检测率均达到 **100.0%**。
  - 每 10 秒闪烁次数为 0，抗干扰能力极佳。
  - 侧向偏移标准差：约 0.825 m。
- **Challenging Conditions (弱光/夜晚/复杂路口测试):**
  - 左侧检测率 74.5%，右侧检测率 87.9%，联合检测率 68.3%。
  - 引入鲁棒性策略（如动态调整 S 通道阈值、在 L 通道开启 CLAHE 直方图均衡化）以应对阴影、褪色车道线及眩光。

## 4. Repository Structure
本项目遵循清晰的数据与逻辑分离原则：

```text
.
├── main.py                 # 主程序入口，串联数据流与视频 I/O
├── preprocess.py           # 图像裁剪、色彩空间转换及边缘检测算子
├── pick_warp_points.py     # IPM 标定工具：交互式获取透视变换源坐标
├── warp.py                 # 逆透视变换 (Homography) 核心执行模块
├── lane_fit.py             # 直方图寻峰、滑动窗口与多项式核心解算
├── temporal.py             # 状态机维护、EMA 滤波及置信度评估
├── overlay.py              # 图像逆变换映射、HUD 文字绘制与可视化
├── preview_*.py            # 供开发与调试使用的各个中间环节可视化脚本
└── data/                   # 测试视频目录 (由 .gitignore 忽略)
